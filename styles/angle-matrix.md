# 选题角度矩阵 + 创意维度轮换池（防同质化）

> 多篇内容容易"一个模子"。本表是创意层的轮换原料。`scripts/diversity.py pick` 会从这些池子里
> 抽一组"最近没用过"的组合，`check` 查重（与最近 6 篇 ≥3 维相同即撞车），`record` 记账。
> 品牌层(作者/IP/语气) 由 profile 锁定保持一致，创意层每篇换。

## 一、选题角度（同一个产品，每篇打不同角度）

| 角度 | 切入方式 | 例 |
|---|---|---|
| 痛点 | 从用户的具体难受切入 | "每次月底对账我都想辞职" |
| 数据实证 | 用数字/案例砸 | "实测一周，省了 11 小时" |
| 横向对比 | 和替代品/旧方法比 | "用了它，再回不去 Excel" |
| 真实故事 | 一个完整经历 | "我带团队踩坑那半年" |
| 反对观点 | 唱反调、反共识 | "别再夸 XX 了，它真不万能" |
| 教程拆解 | 手把手怎么做 | "三步把它接进工作流" |
| 场景代入 | 具体场景里它怎么帮 | "周五下班前那一小时" |
| 人群细分 | 只对某类人说 | "一人公司老板才懂的爽" |

## 二、开头钩子池

场景切入 / 数字冲击 / 反常识断言 / 直接提问 / 自嘲自黑 / 对话体 / 悬念前置 / 金句开场

## 三、结构池

清单体 / 故事体 / 对比体 / QA体 / 吐槽体 / 总分体

## 四、标题套路池

情绪词+设问 / 数字清单 / 身份背书 / 痛点+利益 / 反差对比 / 疑问句 / 祈使句

## 五、配图风格池（每篇换，杜绝一眼同源）

写实(风格1) / 扁平信息图(风格2) / HTML卡片(风格3) / 截图标注 / 手绘插画
> 每篇换主色，别老用同一套配色。

## 六、轮换规则

1. 开工前 `diversity.py pick` 拿组合；`check` 复核不撞最近 6 篇（≥3 维相同=撞，撞了重 pick）。
2. 文章风格池可被 profile 的 `style_pool` 限定（如只在 A/B/F 里转）。
3. 生成完成后 `diversity.py record` 记账，供下篇查重。

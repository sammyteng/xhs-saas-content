# 小红书内容合规检测规范

> 发布前扫描正文、标题、标签，分 **P0（红线拦截）** 和 **P1（风险拦截）** 两级。
> P0 命中 → 必须修改；P1 命中 → 同样必须修改。两者均阻断发布，直到全部清零。

---

## P0 · 红线拦截

命中任一条 → **阻断发布**，必须回 STEP 1/2 修改，否则发布后大概率被删帖/限流/封号。

### 1. 诱导互动

明示或暗示「点赞/收藏/评论可换取利益」的话术。

**命中关键词/句式**（含变体）：
- 「点赞+收藏+关注，私信领xxx」
- 「双击❤️/点赞可抽奖」
- 「评论区扣 1 送xxx」
- 「转发/分享给好友可获xxx」
- 「收藏=白嫖」「收藏就是赚到」（带利益暗示时）
- 任何将互动行为与物质/虚拟利益挂钩的表述

**安全表述**：「觉得有用可以收藏」「欢迎评论区聊聊」→ 这些归 P1。

### 2. 免费送/福利引导

以免费/赠送/福利为钩子换取互动或个人信息。

**命中关键词/句式**：
- 「免费领取」「0 元领」「白送」
- 「转发抽送」「评论区送福利」
- 「限时免费」+ 需要互动才能获取
- 「私信发你」「私我免费领」
- 「扫码/加群领福利」

**安全表述**：产品本身免费（「这个工具免费版就够用了」）不算。

### 3. 互赞互关/涨粉

互关互赞、求关注、养号等操作性话术。

**命中关键词/句式**：
- 「互关互赞」「互粉」「互关必回」
- 「求关注」「新号求罩」「关注不迷路」
- 「帮忙点个关注」「粉丝破xxx靠你们了」
- 「每天更新，关注不亏」（强求关注时）

### 4. 违禁词

广告法/平台黑名单词。

**命中关键词（包括但不限于）**：
- 极限词：「最好/最强/最便宜/第一/唯一/全网最/No.1/Top1」
- 绝对化：「100%/永远/绝对/肯定不会」
- 权威伪装：「国家级/官方认证/权威推荐/专家一致认为」
- 医疗相关：「无副作用/根治/药到病除/特效」
- 虚假承诺：「保证赚钱/稳赚不赔/躺着赚」

**安全表述**：「我觉得是目前体验最好的之一」「据说效果不错」→ 加入不确定表达后可通过。

### 5. 站外导流

引导用户离开小红书平台。

**命中关键词/句式**：
- 「加 V/加微信/加 WX：xxx」
- 「QQ 群/微信群：xxx」
- 「点击链接购买/跳转外部」
- 「淘宝/京东/拼多多搜索 xxx」
- 「详情见主页/简介链接」（引导跳转时）
- 任何外部 URL/二维码

**安全表述**：正文提到产品名（「在 xx 官网可以下载」）属于信息陈述，不主动引导，通常安全但归 P1 观察。

### 6. 批量无意义问答

在正文中植入无意义的自问自答刷互动量。

**命中特征**：
- 3 条以上格式雷同的 Q&A（如「Q: xxx？A: 超好用！」重复）
- 明显是为了堆评论/互动量而非真实解答
- 大段复制粘贴的问答模板

**安全表述**：正文中自然穿插 1-2 个 FAQ 属于正常内容。

## P1 · 风险拦截

命中任一条 → **必须修改**，否则有被限流或触发平台审核的高风险。

### 1. 软性引导互动（必须删除）
- 「觉得有用可以收藏一下」
- 「欢迎评论区讨论」「你们觉得呢？」
- 「关注我看更多干货」

### 2. 模糊功效宣称（必须弱化或删除）
- 「效果立竿见影」「用了就回不去了」
- 「秒杀同类产品」「吊打竞品」
- 未列数据/证据的夸张描述

### 3. 对比贬低竞品（必须改为客观对比或删除）
- 「比 XX 好用 100 倍」「XX 简直不能看」
- 未列对比数据的主观贬低
- 可能触发竞品用户举报

### 4. 敏感话题边缘（必须规避或模糊化处理）
- 涉及薪资/裁员/行业负面
- 政策评价/监管话题
- 可能触发平台人工审核的内容

### 5. 外部提及（必须删除或改为模糊描述）
- 提到产品官网（信息陈述，但有导流嫌疑）
- 提到其他平台名称（「B站有教程」等）

## 检测结果格式（写入 content.json）

```json
{
  "compliance": {
    "p0_passed": true,
    "p0_issues": [],
    "p1_passed": true,
    "p1_issues": [],
    "checked_at": "2024-01-01T12:00:00Z"
  }
}
```

## 检测流程

1. **扫描范围**：titles（所有候选）+ body + tags
2. **P0 扫描**：逐条匹配 6 大类红线规则，命中则标记 category + text + location
3. **P1 扫描**：逐条匹配 5 大类风险规则，同样标记
4. **结果写入**：`content.json` 的 `compliance` 字段
5. **阻断判定**：`p0_passed = false` 或 `p1_passed = false` → 返回 STEP 1/2 修改 → 重新检测 → 直到全部通过
6. **两级同等拦截**：P0 和 P1 均必须清零才能继续发布

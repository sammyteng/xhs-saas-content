> 发布前扫描 **标题 / 正文 / 标签 / 配图文字**，分 **P0（红线拦截）** 和 **P1（风险拦截）** 两级。
> P0 命中 → 必须修改；P1 命中 → 同样必须修改。两者均阻断发布，直到全部清零。
>
> 本清单依据小红书现行官方文本蒸馏，每条标注【依据】。规则随平台更新，**以官方最新文本为准**。
> 本工具产出的是 **AI 生成内容**，故 `P0-1 AIGC 标识` 与 `P0-2 虚假体验` 为本场景最高频红线，务必先过。

## P0 · 红线拦截

命中任一条 → **阻断发布**，必须回 STEP 1/2 修改，否则发布后大概率被删帖/限流/封号。

### P0-1 · AIGC 未标识 ⭐【2026 头号红线 · 本工具必触发】

AI 生成或 AI 辅助创作的图文，发布时**必须主动标识**，否则平台据检测强制补标 + 限制分发。

**通过条件（三者齐备）**：
- `content.json` 写入 `ai_disclosure` 字段（如 `"本文由 AI 辅助创作"`）。
- 正文结尾或首图简介带一行 AI 声明（人类可读的显式标识）。
- 交付摘要提醒用户：发布时勾选【设置 → 内容类型声明 → 笔记包含 AI 合成内容】。

**命中（不通过）**：`ai_disclosure` 缺失，或正文/简介无任何 AI 声明。
**【依据】**《AIGC 标识办法》2025-09-01（显式+隐式标识，禁删改伪造）；薯管家 2026-02-12 公告（未主动标记 → 强制补打标识 + 限制分发）。

**责任边界（依《AIGC 标识办法》）**：

① **隐式标识**（文件元数据 / C2PA / 数字水印）属**生成服务提供者**（即 Gemini / OpenAI / 豆包 / 通义万相 等生图平台）的义务，由平台出图时写入；本工具对其**原样透传保留、不剥离**。

② 本 skill 用户是**内容发布者**，义务仅为**主动显式声明 + 使用平台标识功能**，即 P0-1 现有三项通过条件已完整覆盖。

③ 故配图的**隐式标识不由本 skill 负责**、图面也**不强制可见水印**；平台级「笔记包含 AI 合成内容」声明**整篇覆盖（含图）**。

### P0-2 · 虚假体验 / 虚构人设 ⭐【虚假种草红线 · 本工具高频】

不得编造未真实发生的**使用经历、消费经历、身份战绩、效果数据**。本工具据产品卖点生成内容，第一人称"亲历"措辞极易踩此线。

**命中关键词/句式**：
- 第一人称假装亲历：「我用了三个月」「亲测一周」「实测下来」——但内容实为据资料生成、无真实使用支撑。
- 虚构身份/战绩：「清华毕业」「做了三年」「月入 10 万」「带过 50 人团队」等未经核实的人设。
- 编造效果数字：「转化率翻了 3 倍」「省了 11 小时」等无真实来源的具体数据。

**安全表述**：改为不假装亲历的客观口径——「据官方介绍」「从功能上看」「如果按描述」「以下为基于产品资料的整理」；确有真实体验再用第一人称，且数据须可追溯。
**【依据】**公约 2.0 第 3 条（不塑造虚假人设）、第 20 条（不虚构功效/不伪造数据）；啄木鸟虚假种草治理（2025 起"无科学依据和对比的笔记"纳入虚假营销）。

### P0-3 · 诱导互动

明示或暗示「点赞/收藏/评论可换取利益」的话术。

**命中关键词/句式**（含变体）：
- 「点赞+收藏+关注，私信领 xxx」「双击❤️/点赞可抽奖」「评论区扣 1 送 xxx」
- 「转发/分享给好友可获 xxx」「收藏=白嫖」「收藏就是赚到」（带利益暗示时）
- 任何将互动行为与物质/虚拟利益挂钩的表述。

**安全表述**：「觉得有用可以收藏」「欢迎评论区聊聊」→ 归 P1-1。

### P0-4 · 免费送 / 福利引导

以免费/赠送/福利为钩子换取互动或个人信息。

**命中关键词/句式**：「免费领取」「0 元领」「白送」「转发抽送」「评论区送福利」「限时免费」（需互动才能获取）、「私信发你」「私我免费领」「扫码/加群领福利」。
**安全表述**：产品本身免费（「这个工具免费版就够用了」）不算。

### P0-5 · 互赞互关 / 涨粉

互关互赞、求关注、养号等操作性话术。

**命中关键词/句式**：「互关互赞」「互粉」「互关必回」「求关注」「关注不迷路」「帮忙点个关注」「粉丝破 xxx 靠你们了」「每天更新，关注不亏」（强求关注时）。

### P0-6 · 违禁词 / 极限词

执行《广告法》第九条 + 平台黑名单词。⚠️ 小红书**未公开过官方违禁词表**，以下为通行整理，仅供参考。

**命中关键词（含但不限于）**：
- 极限词：「最好/最强/最便宜/第一/唯一/全网最/No.1/Top1/顶级/国家级/独一无二/绝无仅有」
- 绝对化：「100%/永远/绝对/肯定不会/百分百根治」
- 权威伪装：「官方认证/权威推荐/专家一致认为」
- 医疗相关：「无副作用/根治/药到病除/特效」
- 虚假承诺：「保证赚钱/稳赚不赔/躺着赚」

**资质标注（连带要求）**：无资质**勿给医疗/投资/购房建议**；确需涉及，须显著标「不涉及医疗/投资建议」。
**安全表述**：「我觉得是目前体验最好的之一」「据说效果不错」→ 加不确定表达后可通过。
**【依据】**《广告法》第九条；公约 2.0 第 9 条（无资质勿给医疗/投资/购房建议）。

### P0-7 · 站外导流

引导用户离开小红书平台。**2025-03-12《交易导流违规管理细则》全场景覆盖**：笔记/评论/私信/资料/合集/群聊均算。

**命中关键词/句式**：
- 联系方式（含谐音）：「加 V/加微信/加 wx/V 心/薇/➕我」「QQ 群/微信群：xxx」「邮箱 xxx」
- 站外跳转：「点击链接购买」「淘宝/京东/拼多多搜索 xxx」「详情见主页/简介链接」
- 隐晦话术：「私信了解优惠」「加好友享折扣」「懂的扣 6」
- 任何外部 URL / 二维码 / 口令 / ID 水印。

**安全表述**：正文提到产品名（「在 xx 官网可以下载」）属信息陈述，不主动引导，通常安全但归 P1-5 观察。
**【依据】**社区规范 3.2 导流条款；《交易导流违规管理细则》2025-03-12（合规通道仅企业专业号「私信获客工具」+ 官方留资组件）。

### P0-8 · 批量无意义问答

在正文中植入无意义的自问自答刷互动量。

**命中特征**：3 条以上格式雷同的 Q&A（如「Q: xxx？A: 超好用！」重复）、明显为堆互动量而非真实解答、大段复制粘贴的问答模板。
**安全表述**：正文自然穿插 1-2 个 FAQ 属正常内容。

### P0-9 · 垂类准入红线（条件触发）🆕

**若产品/内容涉以下类目 → 无资质禁碰**：

- **医疗 / 医美**（2026-02 史上最严，从限营销升级为禁准入）：禁推荐具体医疗产品/项目、禁效果证明/案例/证言；处罚永久封号/关播/关商品分享。
- **金融**：未认证禁荐股/基金/房产分析；投资理财产品列违禁物品。
- **教育培训**：未认证禁学科类培训。
- **烟酒**：烟草列违禁物品。

**【依据】**《禁发商品及信息管理规范》2026-02-05、《蒲公英医疗医药管控公告》2026-01-30；社区规范 2.3 违禁物品。

## P1 · 风险拦截

命中任一条 → **必须修改**，否则有被限流或触发平台审核的高风险。

### P1-1 · 软性引导互动（必须删除）
「觉得有用可以收藏一下」「欢迎评论区讨论」「你们觉得呢？」「关注我看更多干货」。

### P1-2 · 模糊功效宣称 / 无对比数据（必须弱化或删除）
「效果立竿见影」「用了就回不去了」「秒杀同类产品」「吊打竞品」；未列数据/证据的夸张描述，或"无科学依据和对比"的功效断言。
**【依据】**公约 2.0 第 7 条（分享经过科学论证的内容）；虚假营销治理。

### P1-3 · 对比贬低竞品（必须改为客观对比或删除）
「比 XX 好用 100 倍」「XX 简直不能看」；未列对比数据的主观贬低（可能触发竞品用户举报）。
**【依据】**公约 2.0 第 24 条（不贬低竞争对手，不编造不实信息）。

### P1-4 · 敏感话题边缘（必须规避或模糊化）
涉薪资/裁员/行业负面、政策评价/监管话题、易触发人工审核的内容；制造性别/地域/职业对立的表述。
**【依据】**公约 2.0 第 13 条（尊重不同性别/地域/职业，反对制造对立）。

### P1-5 · 外部提及（必须删除或改为模糊描述）
提到产品官网（信息陈述但有导流嫌疑）、提到其他平台名称（「B 站有教程」等）。

### P1-6 · 软广特征 🆕（必须改为真实体验向）
产品与内容无相关性、无真实体验支撑、过分参照商家指引、**堆砌官方宣传语句**。这是平台识别"软广"的核心特征，本工具据卖点生成时极易堆官方话术。
**改法**：把"官方宣传语"翻成真人体感（参见 `writing-deai.md` 与 `product-discovery.md` 的「产品语言→小红书语言」转化表）。
**【依据】**社区规范 4.1.4（软广特征）。

### P1-7 · 标题党 / 文题不符 🆕（必须改）
用夸张、猎奇手段做标题，但正文撑不起标题承诺；首图/标题与正文不符。
**【依据】**社区规范 4.1.2；公约 2.0 第 6 条（不做标题党，文题相符）。

### P1-8 · 炫富 / 夸大消费（必须弱化）
炫耀远超常人的消费能力、夸大财富（"是否对别人有用"是判定炫富的标准）。SaaS 内容较少触及，但"一人公司月入百万"类叙事需谨慎。
**【依据】**公约 2.0 第 5 条。

## 检测结果格式（写入 content.json）

```json
{
  "compliance": {
    "ai_disclosure": "本文由 AI 辅助创作",
    "p0_passed": true,
    "p0_issues": [],
    "p1_passed": true,
    "p1_issues": [],
    "checked_at": "2026-01-01T12:00:00Z"
  }
}
```

- `ai_disclosure`：AI 声明文案（P0-1 必填，空字符串视为未通过）。
- `p0_issues` / `p1_issues` 元素格式：`{ "rule": "P0-2", "text": "命中的原文", "location": "title|body|tags|image_text", "fix": "建议改法" }`。其中 `image_text` 指配图上印的文字（统一用 `image_text` 这个词）。

## 检测流程

1. **扫描范围**：titles（所有候选）+ body + tags + 配图内嵌文字。
2. **P0 扫描**：逐条匹配 P0-1~P0-9（AIGC 标识、虚假体验、诱导互动、免费送、互赞互关、违禁词、站外导流、批量问答、垂类准入），命中标记 `rule + text + location + fix`。
3. **P1 扫描**：逐条匹配 P1-1~P1-8，同样标记。
4. **结果写入**：`content.json` 的 `compliance` 字段（含 `ai_disclosure`）。
5. **阻断判定**：`p0_passed = false` 或 `p1_passed = false` → 返回 STEP 1/2 修改 → 重新检测 → 直到全部通过。
6. **两级同等拦截**：P0 与 P1 均须清零才能交付。

# 发布前质量检查清单

> 执行完毕前逐项确认，P0 必须全过，P1 强烈建议，P2/P3 按情况判断。

## P0 — 致命（不过不能发布）

- [ ] **卡片尺寸**：每张 `1080px × 1350px`，无一例外
- [ ] **卡片 ID**：`id="card1"` 到 `id="cardN"` 连续无跳号
- [ ] **截图完整**：首张（封面）和末张（总结）经视觉验证无截断/溢出
- [ ] **文字可读**：标题≥56px，正文≥24px，无文字被裁切
- [ ] **截图脚本**：使用自动探测卡片数（`page.locator`），无硬编码 `range(1,7)`
- [ ] **HTML 注释安全**：无 `═══` `──` 等连续 `--` 的注释格式
- [ ] **色彩匹配**：强调色与背景对比度足够，深色背景下文字清晰

## P1 — 重要（强烈建议修复）

- [ ] **张数合理**：6-9 张，与观点数量匹配（≤3 观点→6 张，4-5→7-8，≥6→9）
- [ ] **首尾固定**：Card 1 = 封面，Card N = 总结，中间为弹性内容
- [ ] **底部导航**：每张有 `.card-footer` + `.prog-dot`，当前页高亮
- [ ] **文案字数**：800-1200 字区间（过短收藏率低，过长完成率低）
- [ ] **标题公式**：15-25 字，含数字或 emoji，有悬念/痛点
- [ ] **话题标签**：5-10 个，大流量词 + 垂直词混搭
- [ ] **飞书图片**：`--file` 使用相对路径，先 `cd` 到目录
- [ ] **飞书权限**：Bot 创建后已为用户授权 `full_access`

## P2 — 优化（提升品质）

- [ ] **CSS 变量化**：所有颜色走 `:root` 变量，无硬编码 hex
- [ ] **品牌匹配**：涉及已知品牌时，已加载对应设计系统
- [ ] **组件统一**：标签/引用块/数据行/步骤流程用 SKILL.md 标准组件
- [ ] **Emoji 克制**：每段≤2 个，用于段落类型标记，非装饰
- [ ] **来源标注**：文案末尾有 `📌 来源：` 信息

## P3 — 锦上添花

- [ ] **设计探索**：用户需求模糊时，出了 3 个方向的封面样品
- [ ] **视觉验证**：用 `vision_analyze` 验证了截图效果
- [ ] **文案打磨**：句子≤20 字，无"然而/因此/综上所述"等论文用语
- [ ] **交叉引用**：文案中图号（图1~图N）与实际卡片数一致

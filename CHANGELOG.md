# 更新说明 / Changelog

本文件记录 `xhs-saas-content` 的版本变化。最新在上。

## v0.2.0 — 通用化大改版（2026-06-14）

> **一句话**：从「混着浏览器自动化的半成品」重构为「**通用图文内容生成器**」——默认产出**含图成品**（文案 + 封面 + 配图）；有 key 用 AI 图、没 key 用 HTML 卡片（免 key、中文零错字），谁下载都能跑出成品。

### ⚠️ 重大变化（升级须知）

- **移除浏览器自动化层**：删掉 fork 自上游的 `scripts/xhs/`、`cli.py`、`bridge_server.py` 及 `skills/` 下 6 个浏览器操作子 skill。本 skill **只生成内容、不自动发布、不操作账号**。
- **默认产出含图成品**：交付完整图文（文案 + 封面 + 1-9 配图），不是半成品。配图走降级链——有 key 用 AI 图、没 key 用 HTML 卡片（免 key、中文零错字），任何环境都出真图；可问偏好但不卡流程，不答即由 AI 判断默认直接产出。
- **交付物简化**：从「编辑版 + 分享版」两份预览改为 **`content.json` + 单个只读模拟器**（左图右文、单文件、图片内嵌）。
- **流程顺序修正**：改为**先写文案、再做图**（修复了旧版「先做封面 → 图配不上文案」的顺序错误）。

### ✨ 新增

- **内容来源 3 模式**（开工第一步三选一）：① 知识库提取（给文件夹/文件/文本）② 直接给 ③ 历史自动出题（记品牌 + 看历史，自动出不重复的新选题）。
- **2026 内容合规检测**：依官方现行文本（社区公约 2.0 / 社区规范 / AIGC 标识办法 / 交易导流细则等）蒸馏为 **P0 红线 9 类 + P1 风险 8 类**，每条带官方依据。含 AIGC 标识（默认关闭，可一键开启）、虚假体验/虚构人设、垂类准入（医疗医美/金融/教培/烟酒）等。图上印的文字也会复检。
- **「别招骂」立意红线**：不站到读者对立面、不唱衰/取代某职业（AI 是帮手不是替代），避免「全交给 AI/淘汰 XX」这类招骂表达。
- **文案体系**：8 种文章风格（A–H）+ 每种字数区间 + 标签生成策略（tag_strategy）+ 品类泛化（不锁科技腔）+ 去 AI 味清单。
- **封面生成器并入 `cover/`**：原 `xhs-cover-skill` 整个融入，22 种人物封面风格自包含；用户自配 key 即用，**仓库零 key**。
- **HTML 卡片风格库**：6 款无生图也能还原的卡片风格（克制蓝/暖色出版/Swiss 黑白/暗色双色/撞色粗野/黑白大字报）。
- **多模型生图**：默认 image-2（OpenAI `gpt-image-2`），可切 gemini / 豆包即梦 / 通义万相，或 HTML 卡片兜底（免 key、中文零错字）。

### 🛡️ 修复 / 健壮性

- 模拟器修复 2 处脚本注入（XSS）隐患；本地编辑服务加 Origin/Host 校验（防 CSRF/DNS-rebinding）。
- 生图脚本加网络超时 + 失败重试；保存失败不再误报成功。
- 封面与内页配色联动（design-token）保留；配图自检负面清单补充人物畸形/乱码字/厂商水印等。

### 🔑 配置须知（默认就出图；想用 AI 图才需要 key）

- **出图必装 playwright**（HTML 卡片出图，无 key 的兜底方式）：`pip install playwright && playwright install chromium`。
- **AI 图（可选，更丰富）**：纯设计封面/内页配 `OPENAI_API_KEY`（+ `--base-url` 接兼容聚合站），默认模型 `gpt-image-2`；人物照封面用 `cover/`，首次 `cd cover && npm install`，自配 chat-image key（`XHS_COVER_API_KEY` 等）。
- **没 key 也出成品**：直接走 HTML 卡片。**所有 key 仅从环境变量 / 用户本地 config 读取，仓库不含任何 key。**
